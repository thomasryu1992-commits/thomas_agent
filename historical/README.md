# Historical Artifacts

**Authority:** None
**Runtime use:** Prohibited

This directory preserves superseded architecture, frozen phase evidence, migration review records, and retired compatibility implementations. Historical material may be used for audit, migration, or reproducibility review, but it is never a current Runtime or Governance source.

The canonical classification is [`HISTORICAL_ARTIFACT_INDEX.yaml`](HISTORICAL_ARTIFACT_INDEX.yaml).

Core release manifests and their copied source/toolchain snapshots remain in their original release directories because their paths and hashes are part of immutable release evidence. The Historical index classifies them without rewriting them.
